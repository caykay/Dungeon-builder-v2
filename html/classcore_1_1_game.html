Detailed Description

The Game class This class employs the singleton pattern, meaning there can only be one of its instance at a time.

It is responsible of monitoring and keeping track of any Dungeon builders and levels. Its constructor is set to private, such that no object of Game can be instantiated. Also any copy constructor of Game will be deleted if ever made

Author
Cassian Kahema
